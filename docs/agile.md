# Investigación: Metodologías de Desarrollo

## 1. ¿Qué es Agile?

Agile es una filosofía de trabajo que busca entregar software de calidad de forma rápida y constante. En lugar de planificar todo el proyecto durante meses y entregarlo al final, Agile propone dividir el trabajo en pequeñas partes funcionales. Su objetivo principal es la **adaptabilidad**: poder reaccionar a cambios o errores sin tener que empezar de cero.

## 2. Conceptos Clave
 ### Scrum
Es un marco de trabajo rígido pero efectivo. Se basa en:
* Roles: Product Owner, quien tiene la idea, Scrum Master, quien quita obstáculos, y el Equipo de Desarrollo.

* Sprints: Ciclos de trabajo cortos donde se completa una lista de tareas.

* Backlog: La lista maestra de todo lo que la app necesita.

 ### Kanban

Es un método visual para gestionar el trabajo a medida que avanza. Se usa un tablero con columnas (To Do, Doing, Done). Su objetivo es el flujo continuo y evitar que el equipo se sature limitando el trabajo en curso (WIP).

## 3. Diferencias Principales

* Scrum tiene tiempos fijos  y roles definidos; Kanban es un flujo continuo sin fechas de inicio/fin para las tareas.

* Scrum no permite cambios durante un Sprint; Kanban permite añadir tareas nuevas en cualquier momento si hay espacio en la columna.

## 4. Aplicación en Batnie

Para el desarrollo de Batnie, utilizaré una metodología inspirada en Kanban. 

* Al ser un proyecto individual y creativo, Kanban me permite visualizar el progreso de las funcionalidades (como el Grid o la API) y moverlas en el tablero según las vaya completando, sin la presión de los Sprints cerrados de Scrum. Es ideal para mantener el "flujo creativo" que la propia app propone.
